博客迁移到github pages

前言

因为快要毕业了，如果再用阿里云估计也要用不起了，很早就想把博客放在github上，免费的同时访问速度也还行。

博客文章备份

之前的博客是放在云主机上的，所以下到了本地。主要备份了source文件夹（主要是文章和个人简介）、themes中的主体文件夹，和_config.yml(站点的，主题有自己)的。
不得不称赞hexo很简洁，备份没有花多少功夫。

本地安装nodejs git

nodejs下载地址
git下载地址
傻瓜式安装，没有什么要注意的。

hexo安装

运行git bash
npm install hexo -g

记得cd到一个你要放博客的文件夹再初始化，笔者刚开始直接hexo init，结果就初始化到C:\Users\shinelin里了。 或者直接hexo init
比如我放在了f盘，cd f://hexo_blog。

拷贝博客

直接把原来的备份覆盖过来就行，不过中间有几个问题。

  • 需要安装一下feed sitemap npm install hexo-generator-feed --save、npm install hexo-generator-sitemap --save。
  • 由于我使用了bootstrap的标签，所以要npm install npm install hexo-tag-bootstrap --save
  • _config.yml里面一些没有的插件给删除了，不然可能会报错。（我很奇怪我安装了feed为什么会报错。。。）

浏览器可以打开了，完美～～

部署到github

在github上Create a new repository
然后Repository name填写你的用户名.github.io,例如shinelinxx.github.io
其他设置为默认

先安装部署工具

本地修改(添加)_config.yml (站点目录下的)， repository就是你源的ssh的地址，在github上复制。

这时候你会发现hexo d部署会提示没有权限，这是因为你需要配置ssh.

配置ssh

参考这篇博客
你会发现在第四步的时候可能还是连接不上，能连上最好。
连不上的话，看这里
里面的博主用vim编辑，在windows下你完全可以用记事本。

注：publickey别自作多情改成你的ssh key了，我之前就是。

到这里应该就可以连接上了。
接着hexo d， 在github上刷新,好了部署上去了.
用shinelinxx.github.io访问，也能打开了。
